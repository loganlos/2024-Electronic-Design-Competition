
# 2024年全国大学生电子设计竞赛 - H题：自动行驶小车

本仓库包含 2024 年全国大学生电子设计竞赛（TI 杯）H 题——**自动行驶小车**的全套控制源代码。项目基于 **TI MSPM0** 系列微控制器，实现了高精度的路径识别、姿态控制及多任务逻辑切换。


## 📌 项目概述

本项目要求小车在指定的白色背景、黑色线条场地上，根据预设的不同模式完成自动行驶任务。核心挑战在于不使用摄像头的前提下，仅依靠传感器实现精准的寻线、点位判定及闭环控制。

## 🛠️ 硬件架构

* **主控芯片**：TI MSPM0G3507 (MSPM0 系列) 。
* **传感器组**
* **寻线模块**：8 路红外/灰度传感器 (P1-P8)，用于路径识别与位置校准。
* **姿态感知**：JY61P 陀螺仪 (串口通信)，提供高频率的 Yaw 角反馈。
* **速度反馈**：霍尔编码器，实现双轮速度闭环。
* **执行器**：直流减速电机，通过 PWM 驱动实现差速转向。

## 📂 代码结构

| 文件名 | 描述 |
| --- | --- |
| `main` | 系统初始化、中断配置及任务调度核心逻辑。 |
| `control` | 核心控制算法：包含增量式 PID、陀螺仪偏航角控制及 PWM 限幅。 |
| `Sensor` | 路径决策引擎：处理 8 路寻线信号，实现点位判定 (A/B/C/D) 及状态机切换。 |
| `Encoder` | 编码器信号采集与单位时间速度换算。 |
| `GYRO` | 陀螺仪协议解析，通过 UART 异步接收解析欧拉角数据。 |

## 🚀 核心算法实现

### 1. 增量式 PID 控制

项目采用了典型的双环控制架构：

* **速度环**：通过编码器反馈，确保左右电机转速稳定，补偿电池电压波动带来的影响。
* **偏航环 (Yaw)**：在无黑线段或特殊转弯段，利用陀螺仪角度进行辅助导航。

### 2. 状态机逻辑

在 `Follow_Route()` 函数中，通过 `flag` (段位) 和 `mode` (任务) 构成的二维状态机，精准捕捉 A/B/C/D 各个顶点的特征信号（如全白、全黑或特定侧检测），并触发相应的声光提示。

### 3. 动态寻线加权

根据 8 路传感器位置设置权重系数，将离散的开关量转化为连续的偏差值 `bias`，提供给转向环，使小车行驶轨迹丝滑、无摆动。

## 💻 快速开始

1. **环境配置**：使用 TI Code Composer Studio (CCS) 或 Keil MDK (配合 MSPM0 SDK)。
2. **编译烧录**：将工程导入 CCS，通过仿真器烧录至开发板。
3. **操作指南**：
* 按下 `KEY_S2` 切换模式 (1-4)。
* 按下 `KEY_EN` 使能控制系统，小车开始运行。
* 观察 LED 指示灯及蜂鸣器，判断是否经过顶点或完成任务。



## 📈 性能表现

* **稳定性**：引入陀螺仪后，小车在长距离直线行驶中几乎无漂移。
* **响应速度**：控制频率设定在 10ms (100Hz)，能够满足高速行驶下的瞬态响应要求。
